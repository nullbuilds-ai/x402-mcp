Before / After

x402 without the ceremony.

Handling x402 manually means managing wallets, EIP-3009 signing, retries, and payment headers. x402-mcp does it in one tool call.

Without x402-mcp
// 1. Probe the 402
const probe = await fetch(url);
// 2. Parse payment requirements
const req = parseX402Header(probe);
// 3. Sign EIP-3009 transfer
const sig = await signEIP3009(wallet, req);
// 4. Encode payment header
const header = encodeHeader(sig);
// 5. Retry with payment
const res = await fetch(url, {
  headers: { 'X-PAYMENT': header }
});

FAQ
Do I need a crypto wallet to use this?
No. discover_paid_services, estimate_payment, and check_wallet_balance work with zero credentials. You only need a wallet to make actual payments via make_x402_request.
Is my private key on my machine?
No. Payments use Coinbase CDP Server Wallets — signing happens in Coinbase's TEE. Your private key never touches your device.
How is the catalog kept up to date?
A cron job probes all 13,334 registered x402 endpoints nightly. Only the 957 that respond correctly make it into the catalog.
What MCP clients does this work with?
Any MCP client: Claude Desktop, Cursor, Windsurf, and anything else that supports the Model Context Protocol.
